# Astra Studio Landing Page

## 📌 Project Overview

This project is a **production-focused frontend landing page** built using **HTML and CSS only**. The goal was not just to design a UI, but to **simulate a real-world development workflow** where code quality, structure, and maintainability are as important as the final design.

The project was developed under the guidance of **ChatGPT acting as a client / senior frontend developer**, who assigned a **realistic development task** and reviewed the work based on industry-level expectations.

---

## 🎯 Objective

The primary objective of this project was to:

* Build a clean and modern landing page
* Apply **BEM (Block Element Modifier)** methodology
* Organize assets and files in a professional structure
* Write **clean, scalable, and maintainable CSS**
* Prepare the project for **GitHub and production readiness**

---

## 🧠 Development Approach

Instead of following tutorials or copying designs, this project was built through:

* A **command-based workflow** (like real client tickets)
* Independent problem solving
* Iterative improvements based on feedback
* Refactoring and cleanup phases after initial development

---

## 🚧 Challenges That I Faced...

This project was not just about writing code — it involved solving real frontend problems that developers face in production environments.

### 1. **Understanding and Applying BEM Properly**

Initially, class naming became overly complex and confusing.
Learning to simplify BEM into a **clean and reusable system** was a major challenge.

### 2. **Avoiding Over-Engineering**

There was a tendency to create long and complicated class names.
The challenge was to shift toward **readability and simplicity over complexity**.

### 3. **CSS Cleanup and Organization**

Managing:

* Duplicate styles
* Unused CSS
* Inconsistent spacing

Refactoring the CSS into a **structured and scalable format** required careful thinking.

### 4. **Consistency System**

Maintaining consistent:

* spacing
* typography
* colors

was difficult at first, but essential for a professional UI.

### 5. **File and Folder Structure**

Learning how to organize:

* assets
* CSS files
* project hierarchy

in a clean and production-ready way was a key learning point.

### 6. **Semantic HTML Decisions**

Choosing between:

* `<button>` vs `<a>`
* `<div>` vs `<section>`

and applying correct semantics was an important challenge.

### 7. **Thinking Like a Developer, Not a Student**

The biggest shift was moving from:

> “Just make it work”

to:

> “Make it clean, scalable, and understandable for other developers”

---

## 💡 Key Learnings

* Clean code is more valuable than complex code
* Consistency is more important than creativity
* Naming conventions define project readability
* A reusable system (like BEM) saves time in future projects
* Real growth happens during **refactoring and cleanup**, not just building
