# Cross-Device Layout Project

## Project Overview

This project is a **responsive landing page section** built entirely with **HTML and CSS**.  
The goal was to **practice cross-device testing and fix edge-case layouts** across desktop, tablet, and mobile screens. The project focuses on **layout stability, flexbox/grid handling, spacing, and responsive edge cases**.

---

## Client / Senior Developer

This project was assigned as a **realistic development task** by my **AI Senior Developer** (simulating a professional client/lead).  
I received the project brief and **built the entire layout independently** based on the instructions.

### Key Instructions Provided by AI (as client/senior dev):

- Create an **announcement bar** with text that never overflows on small screens.  
- Build a **navigation bar** with logo and menu that adapts from desktop to mobile without breaking.  
- Design a **hero section** introducing the AI Dashboard feature, with proper headings, paragraphs, buttons, and an image, responsive across devices.  
- Add a **feature cards section** with 3 cards, handling spacing, wrapping, and height consistency for all screen sizes.  
- Include a **testimonial section** with avatar and quote, ensuring it works on all devices.  
- Handle **edge-case layout problems** such as long text, image overflow, button wrapping, and flexbox collapse.  
- Test the layout across **desktop, tablet, and mobile screen widths** to ensure **layout robustness**.

> “Your job is to build a production-ready layout that survives all edge cases across devices. Think like a professional frontend developer testing for real-world scenarios.” — AI as Senior Developer

## How This Project Was Built

1. Received a **project brief from AI** (acting as a client/senior dev).  
2. Designed the **layout section by section**, ensuring responsiveness.  
3. Fixed layout **edge cases** such as:
   - Long text in announcement bar  
   - Navigation overflow on mobile  
   - Hero buttons and image alignment  
   - Card spacing and wrapping  
4. Tested on **multiple screen widths**: 1440px, 1024px, 768px, 480px, and 360px.  
5. Implemented a **clean CSS structure** with variables and consistent spacing.
